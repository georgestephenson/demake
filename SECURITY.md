# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities **privately** through GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability):
open the repository's **Security** tab → **Report a vulnerability**.

Please do not open public issues for security problems.

We aim to acknowledge reports within a few days and will keep you updated as we
investigate and prepare a fix.

## Supported versions

The project is pre-1.0 and under active development. Until `v1.0.0`, only the
latest release receives security fixes.

| Version | Supported      |
| ------- | -------------- |
| `0.x`   | ✅ latest only |
